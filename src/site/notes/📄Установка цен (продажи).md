## 1. Назначение

Документ **Установка цен продажи** фиксирует новые цены на товары и записывает их в регистр сведений **Цены продажи**.

После **проведения** документа цены начинают действовать с указанной **даты** для выбранного **склада**. Их используют документы продажи, отчёты и внешние интеграции.

**Когда создавать документ:**

- при изменении розничных или оптовых цен;
- при переоценке ассортимента на конкретном складе;
- при массовом обновлении цен после изменения закупочной стоимости или курса валюты.

> **Важно:** цены привязаны к **складу**. Один и тот же товар на разных складах может иметь разные цены — для каждого склада создаётся отдельный документ (или отдельное проведение с нужным складом в шапке).

---

## 2. Где найти

**Раздел:** Продажи → **Установка цен продажи**

В списке отображаются номер, дата, склад, автор и состояние документа (проведён / не проведён).

---

## 3. Шапка документа

| Поле | Описание |
|------|----------|
| **Номер** | Присваивается автоматически |
| **Дата** | Дата начала действия цен. При проведении записи в регистре получают этот период |
| **Кем создан** | Автор документа (только просмотр) |
| **Склад** | Склад, для которого устанавливаются цены. При создании нового документа подставляется склад текущего сеанса пользователя |
| **Примечание** | Произвольный комментарий (причина переоценки, номер приказа и т.п.) |

---

## 4. Пошаговый сценарий работы

### Шаг 1. Создать документ

1. Откройте список **Установка цен продажи**.
2. Нажмите **Создать**.
3. Проверьте **дату** и **склад** в шапке.

### Шаг 2. Настроить виды цен (вкладка «Настройки»)

На вкладке **Настройки** отметьте галочками виды цен, которые нужно изменить (например, *Мелкий опт*, *Крупный опт*).

- При создании документа по умолчанию отмечены все неудалённые виды цен.
- Снятие галочки скрывает соответствующие колонки на вкладке **Товары** — цены этого вида в документ не попадут.

При необходимости укажите **Количество порогов цен** (от 0 до 3) — это включает ступенчатые цены в зависимости от количества (подробнее — раздел 5).

### Шаг 3. Заполнить табличную часть «Товары»

Добавьте номенклатуру одним из способов (раздел 6) и укажите новые цены в колонках выбранных видов цен.

Для каждого вида цен в таблице отображаются:

- **Цена старая** — текущая действующая цена (только просмотр);
- **Цена новая** — базовая цена (Цена0), которую нужно ввести;
- при включённых порогах — пары **от количества** / **цена** для скидок при больших объёмах.

Дополнительные справочные колонки (включаются на вкладке «Настройки»):

- **Спецификация** — флаг/страна из свойств номенклатуры;
- **Остаток** — остаток на складах (клик открывает отчёт по остаткам);
- **Закупка** — последняя цена закупки (клик открывает обработку «Цены поставщиков»).

### Шаг 4. Проверить и записать

1. Убедитесь, что в таблице **нет дублей** — одна номенклатура может быть только в одной строке.
2. Нажмите **Записать**.

При попытке записи с дублями система покажет сообщение с номерами строк.

### Шаг 5. Провести документ

Нажмите **Провести** (или **Провести и закрыть**).

После проведения:

- в регистре **Цены продажи** появятся записи по каждой комбинации *номенклатура + вид цены + склад*;
- цены с более поздней датой не перезаписываются — действует принцип «последний документ по дате».

### Шаг 6. Отмена проведения (при необходимости)

**Отменить проведение** — удаляет записи этого документа из регистра. Используйте, если документ проведён ошибочно.

> Перед отменой проведения убедитесь, что нет более поздних документов установки цен по тем же товарам, складу и виду цены — иначе актуальными останутся цены из более позднего документа.

## 5. Пороговые (ступенчатые) цены

Пороговые цены позволяют задать **разную цену продажи в зависимости от количества** в заказе. Это удобно для оптовых продаж: чем больше покупатель берёт, тем ниже цена за единицу.

### 5.1. Как включить

1. Откройте вкладку **Настройки**.
2. В поле **Количество порогов цен** укажите нужное значение — от **0** до **3**:
   - **0** — только базовая цена (без ступеней);
   - **1** — одна дополнительная ступень (Порог1 / Цена1);
   - **2** — две ступени;
   - **3** — три ступени (максимум).

После изменения значения в таблице **Товары** появятся или скроются соответствующие колонки.

### 5.2. Поля в таблице

Для каждого **вида цены** (группа колонок, например «Мелкий опт ($)»):

| Поле | Назначение |
|------|------------|
| **Цена новая** | Базовая цена — применяется, если количество **меньше** первого порога |
| **от количества** (Порог1) + **цена** (Цена1) | Первая оптовая ступень |
| **от количества** (Порог2) + **цена** (Цена2) | Вторая ступень (больший объём — ниже цена) |
| **от количества** (Порог3) + **цена** (Цена3) | Третья, наиболее выгодная ступень |

Колонка **Цена старая** показывает текущую действующую базовую цену — только для справки, не записывается как новая.

### 5.3. Как система выбирает цену

При продаже (и при передаче цен во внешние системы) система проверяет пороги **от большего к меньшему**: сначала Порог3, затем Порог2, затем Порог1.

- если количество **≥ Порог3** и **Цена3** заполнена → применяется **Цена3**;
- иначе, если количество **≥ Порог2** и **Цена2** заполнена → **Цена2**;
- иначе, если количество **≥ Порог1** и **Цена1** заполнена → **Цена1**;
- иначе → **Цена новая (Цена0)**.

Порог считается **достигнутым**, если количество в заказе **больше или равно** указанному значению.

### 5.4. Пример: один товар, вид цены «Мелкий опт»

Зададим цены на товар «Кабель HDMI 2 м»:

| Поле | Значение |
|------|----------|
| Цена новая | 120 $ |
| Порог1 / Цена1 | от **5** шт → **110** $ |
| Порог2 / Цена2 | от **20** шт → **100** $ |
| Порог3 / Цена3 | от **50** шт → **90** $ |

**Какая цена применится:**

| Количество в заказе | Цена за единицу | Почему |
|---------------------|-----------------|--------|
| 1 шт | 120 $ | Меньше первого порога (5) → базовая цена |
| 5 шт | 110 $ | Достигнут Порог1 |
| 15 шт | 110 $ | Между Порог1 и Порог2 → действует Цена1 |
| 20 шт | 100 $ | Достигнут Порог2 |
| 50 шт | 90 $ | Достигнут Порог3 |
| 100 шт | 90 $ | Порог3 по-прежнему действует |

### 5.5. Пример: только одна ступень (Количество порогов = 1)

Для розничного вида цены иногда достаточно одной оптовой скидки:

| Поле           | Значение                 |
| -------------- | ------------------------ |
| Цена новая     | 500 $                    |
| Порог1 / Цена1 | от **10** шт → **450** $ |

- заказ 3 шт → **500 $**;
- заказ 10 шт и более → **450 $**.

Порог2 и Порог3 не отображаются и не используются.

### 5.6. Рекомендации по заполнению

- **Пороги должны возрастать:** Порог1 < Порог2 < Порог3 (например: 5 → 20 → 50).
- **Цены должны убывать:** Цена0 ≥ Цена1 ≥ Цена2 ≥ Цена3 — иначе больший объём не выгоднее меньшего.
- **Порог без цены** (цена = 0) — ступень пропускается, система проверит следующий порог ниже.
- **Цена без порога** (порог = 0) — ступень не участвует в расчёте.
- Пороговые цены задаются **отдельно для каждого вида цен** в одной строке: «Мелкий опт» и «Крупный опт» могут иметь разные ступени и значения.

### 5.7. Без порогов (Количество порогов = 0)

Если ступенчатое ценообразование не нужно, оставьте **Количество порогов цен = 0** и заполняйте только **Цена новая**. Это стандартный режим для фиксированной цены на любое количество.

## 6. Команды заполнения табличной части

Команды расположены на панели таблицы **Товары**.
### Заполнить по...

| Команда | Что делает |
|---------|------------|
| **Заполнить по остаткам** | Добавляет товары, которые есть в остатке на выбранном **складе** документа. Товары с запретом отгрузки не включаются. Подтягиваются остатки и справочные данные |
| **Заполнить по товарной группе** | Подменю с группами подбора номенклатуры. Добавляет все товары выбранной группы |
| **Заполнить по бренду** | Подменю с производителями. Добавляет все товары выбранного бренда |
| **Заполнить по группе товаров** | Открывает выбор группы справочника **Номенклатура** и добавляет все позиции из неё |
| **Подбор** | Открывает форму выбора номенклатуры с множественным выбором. Уже добавленные позиции повторно не добавляются |

> Если табличная часть уже содержит строки, при заполнении система спросит: **«Табличная часть не пуста! Очистить?»** — ответ **Да** очистит таблицу перед заполнением, **Нет** — добавит новые строки к существующим (без дублей).

### Прочие команды

| Команда | Что делает |
|---------|------------|
| **Обновить** | Перезагружает справочные данные по уже добавленным строкам: остатки, цену закупки, спецификацию, текущие цены |
| **Удалить незаполненные** | Удаляет строки, в которых не указаны новые цены |
| **Пересчитать** | Открывает форму пересчёта по курсу (поля «Курс было» / «Курс стало»). Пропорционально пересчитывает введённые цены |
| **Пересчитать с наценкой** | Запрашивает процент наценки и рассчитывает **Цена новая** от **Цены старой** по формуле: `старая × наценка / 100` |

## 7. Склад и остатки

### Склад в шапке документа

**Склад** — ключевой реквизит. Именно для этого склада при проведении записываются цены в регистр **Цены продажи**.

Если нужно установить одинаковые цены на несколько складов — создайте отдельный документ для каждого склада (или скопируйте проведённый документ и измените склад и дату).

### Отображение остатков в таблице

На вкладке **Настройки** → группа **Остатки по складам**:

- **Включено (по всем складам)** — в колонке **Остаток** показывается суммарный остаток по всем складам;
- **Выключено** — можно выбрать конкретные склады (ссылка под полем). Остаток считается только по отмеченным складам.

Колонка **Остаток** носит справочный характер и **не влияет** на проведение. Клик по ячейке открывает отчёт **Остатки товара** по этой номенклатуре.

## 8. Типичные ошибки и ограничения

### Дубли номенклатуры

В одном документе **нельзя** указать один товар дважды. При записи система выведет ошибку с номерами строк-дублей.

**Решение:** удалите лишнюю строку или объедините цены в одной строке (для всех выбранных видов цен).

### Повторный выбор номенклатуры в строке

При ручном выборе товара, который уже есть в другой строке, появится предупреждение — выбор будет отменён.

### Запрет записи по дате

Система может блокировать сохранение документов с датой, попадающей в закрытый период (настройка на уровне конфигурации). В этом случае измените дату или обратитесь к администратору.

### Отмена проведения

При отмене проведения записи документа удаляются из регистра **Цены продажи**. Если после этого нужны актуальные цены — проведите новый документ установки цен.


---

## 9. Краткая памятка

```
1. Создать документ → проверить Дату и Склад
2. Настройки → выбрать виды цен, при необходимости пороги
3. Заполнить товары (по остаткам / подбор / группа / бренд)
4. Ввести новые цены
5. Записать → Провести
```

---

## 10. Для продвинутых пользователей

- **Виды цен** настраиваются в справочнике **Виды цен** (название, валюта). Предопределённые: *Мелкий опт*, *Крупный опт*. Новые виды автоматически появятся в документе как дополнительные колонки.
- **История цен** хранится в регистре **Цены продажи** — можно открыть из раздела Продажи для просмотра среза на дату.
- При проведении и отмене проведения система может ставить изменения цен в **очередь WebHook** для отправки во внешние системы (интернет-магазин, API). Это происходит автоматически, если настроены учётные записи интеграции.
