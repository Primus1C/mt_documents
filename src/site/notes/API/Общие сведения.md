### Метаданные

- **Авторизация:** `Bearer Token` (требуются права: `client:write`)
    
- **Content-Type:** `application/json`
    
- **Идемпотентность:** Да, поддерживается через заголовок `X-Idempotency-Key`
### Формат данных

    — запросы с телом: JSON, заголовок Content-Type: application/json

    — ответы: JSON (кроме delivery-notes — бинарный файл)

    — коды справочников (itemCode, warehouseCode, counterpartyCode) — строки: "4", "42"

    — даты в ответах: yyyy-MM-ddTHH:mm:ss

    — даты в query/body: yyyy-MM-dd (из ISO-строки берутся первые 10 символов)

### Пагинация

Используется в balances, counterparties, orders (режим списка).  
  
В запросе отправляем :  
    page  — номер страницы,

    pageSize  — размер страницы, 1–500, по умолчанию 500

    
В ответе всегда :

    "page"  — номер страницы, целое ≥ 1, по умолчанию 1

    "pageSize"  — размер страницы, 1–500, по умолчанию 500

    "totalItems" – общее количество объектов     "totalPages" – общее количество страниц
