> [!important] 
> - orderNo - входящий номер Заказа, изначально задается заказчиком; в дальнейшем идентификация заказа происходит исключительно по этому параметру API-запросов
> - orderNumber - номер 1С, присваивается автоматически при регистрации заказа в 1С

# GET `/api/v1/orders`

**Описание:** получение заказа (заказов)
### Режим Single (один заказ)
**Параметры:** 

| параметр                                 | описание                             | обязательный                                      |
| ---------------------------------------- | ------------------------------------ | ------------------------------------------------- |
| orderNo                                  | номер входящий (нумерация заказчика) | + (см.Hint)                                       |
| counterpartyCode \| customerCode \| code | код заказчика                        | необязателен, если указано наименование заказчика |
| counterparty \| customer \| name         | наименование заказчика               | необязателен, если указан код заказчика           |
| dateFrom                                 |                                      | используется, если не указан orderNo              |
| dateTo                                   |                                      | используется, если не указан orderNo              |
**Запрос**

    GET /trade/hs/stock/orders

**Ответ**

### Режим Multiple (несколько заказов)
**Параметры:** 

| параметр                                 | описание                                                        | обязательный                                      |
| ---------------------------------------- | --------------------------------------------------------------- | ------------------------------------------------- |
| counterpartyCode \| customerCode \| code | код заказчика                                                   | необязателен, если указано наименование заказчика |
| counterparty \| customer \| name         | наименование заказчика                                          | необязателен, если указан код заказчика           |
| dateFrom                                 | дата начала интервала                                           | +                                                 |
| dateTo                                   | дата конца интервала (подразумевается конец дня указанной даты) | +                                                 |
Если не указан параметр, определяющий контрагента, запрос вернет заказы всех контрагентов, входящих в интервал дат.

**Запрос**

    GET /trade/hs/stock/orders?dateFrom=2026-01-01&dateTo=2026-01-31&customer=Ivanoff

    GET /trade/hs/stock/orders?dateFrom=2026-01-01&dateTo=2026-01-31

# POST `/api/v1/orders`

**Описание:** cоздание заказа с резервированием остатков.

**Запрос**

    POST /trade/hs/stock/orders

**Ответ**

# PATCH `/api/v1/orders`

**Название:** замена заказа
**Описание:** замена заказа 

**Запрос**

    PATCH /trade/hs/stock/orders

**Ответ**



> [!attention] 
> - Нельзя изменить заказ, по которому уже создана расходная накладная → ответ 409.
> - Нельзя в заказе указывать цены, не совпадающие с актуальными
> - Нельзя сделать заказ на количество товаров, превышающее остаток склада

# PUT `/api/v1/orders`

**Название:** добавление строк заказа
**Описание:** добавление позиций в существующий заказ (текущие строки не удаляются). 

**Запрос**

    PUT /trade/hs/stock/orders

**Ответ**

# DELETE `/api/v1/orders`

**Название:** удаление строк заказа
**Описание:** удаление позиций из проформы. Если в items перечислены все позиции заказа — документ помечается на удаление. 

**Запрос**

    DELETE /trade/hs/stock/orders
