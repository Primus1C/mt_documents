> [!important] 
> Поиск заказа методами API возможен по одному их параметров запроса (каждый является уникальным идентификатором заказа):
> - `incomingNo | incomingNumber | incoming` - входящий номер Заказа, изначально задается заказчиком
> - `orderNo | orderNumber | order` - номер 1С, присваивается автоматически при регистрации заказа в 1С

> [!attention] 
> - Нельзя изменить заказ (см. [[Статусы Инвойсов продажи\|Статусы Инвойсов продажи]]), по которому уже создана расходная накладная → ответ 409.
> - Нельзя в заказе указывать цены, не совпадающие с актуальными
> - Нельзя сделать заказ на количество товаров, превышающее остаток склада
> - 
# GET `/api/v1/orders`

**Описание:** получение заказа (заказов), зарегистрированного ранее в 1С
### Режим Single (один заказ)
**Параметры:** 

| параметр                                 | описание                             | обязательный                                        |
| ---------------------------------------- | ------------------------------------ | --------------------------------------------------- |
| incomingNo \| incomingNumber \| incoming | номер входящий (нумерация заказчика) | + (необязателен, если указан *номер 1С*)            |
| orderNo \| orderNumber \| order          | номер 1С                             | + (необязателен, если указан *номер входящий*)      |
| counterpartyCode \| customerCode \| code | код заказчика                        | необязателен, если указано *наименование заказчика* |
| counterparty \| customer \| name         | наименование заказчика               | необязателен, если указан *код заказчика*           |
| dateFrom                                 | начало интервала дат заказов         | используется, если не указан  *номер заказа*        |
| dateTo                                   | окончание интервала дат заказов      | используется, если не указан *номер заказа*         |
**Запрос**

    GET /trade/hs/stock/orders

### Режим Multiple (несколько заказов)
**Параметры:** 

| параметр                                 | описание                                                        | обязательный                                      |
| ---------------------------------------- | --------------------------------------------------------------- | ------------------------------------------------- |
| counterpartyCode \| customerCode \| code | код заказчика                                                   | необязателен, если указано наименование заказчика |
| counterparty \| customer \| name         | наименование заказчика                                          | необязателен, если указан код заказчика           |
| dateFrom \| from                         | дата начала интервала                                           | +                                                 |
| dateTo \| to                             | дата конца интервала (подразумевается конец дня указанной даты) | +                                                 |
Если не указан параметр, определяющий контрагента, запрос вернет заказы всех контрагентов, входящих в интервал дат.

**Запрос**

    GET /trade/hs/stock/orders?dateFrom=2026-01-01&dateTo=2026-01-31&customer=Ivanoff

    GET /trade/hs/stock/orders?dateFrom=2026-01-01&dateTo=2026-01-31

# POST `/api/v1/orders`

**Описание:** cоздание заказа с резервированием товаров

**Запрос**

    POST /trade/hs/stock/orders

**Параметры**

| параметр                                 | описание                                                 | обзательный                                               |
| ---------------------------------------- | -------------------------------------------------------- | --------------------------------------------------------- |
| incomingNo \| incomingNumber \| incoming | номер входящий (нумерация заказчика)                     | +                                                         |
| counterpartyCode \| customerCode \| code | числовой код контрагента                                 | + (необязательно, если задано *наименование контрагента*) |
| counterparty \| customer \| name         | наименование контрагента                                 | + (необязательно, если задан *код контрагента*)           |
| warehouseCode \| warehouseId             | числовой код склада                                      | +  (необязательно, если задано *наименование склада*)     |
| warehouse                                | наименование склада                                      | + (необязательно, если задан *код склада*)                |
| employeeCode \| employeePin              | код сотрудника 1С, или его пин                           | -                                                         |
| items                                    | массив сведений о товарах (количество, цена, код товара) | +                                                         |
**Пример запроса**
```json
{
    "incomingNo": "GRI-0130-2",
    "customer": "Global Resell Inc.",
    "warehouse": "Склад Основной",
    "employeePin": "2213",
    "items": [
        {
            "qty": 2,
            "price": 100,
            "itemCode": 23880        
        },
        {
            "qty": 5,
            "price": 90,
            "itemCode": 23881        
        }
    ]
}
```

**Пример ответа**
```json
{
    "success": true,
    "error": "",
    "incomingNumber": "GRI-0130-2",
    "orderNumber": "МТ-20260708-01",
    "orderDate": "2026-07-08T12:13:30",
    "status": "Зарегистрирован",
    "statusSetBy": "Погрузчиков А.П.",
    "statusDate": "",
    "reserved": true
}
```

# DELETE `/api/v1/orders`

**Название:** удаление строк заказа
**Описание:** удаление позиций из проформы. Если в items перечислены все позиции заказа — документ помечается на удаление. 

**Запрос**

    DELETE /trade/hs/stock/orders

**Параметры**

| параметр                                 | описание                             | обязательный                                   |
| ---------------------------------------- | ------------------------------------ | ---------------------------------------------- |
| incomingNo \| incomingNumber \| incoming | номер входящий (нумерация заказчика) | +  (необязателен, если указан *номер 1C*)      |
| orderNo \| orderNumber \| order          | номер 1С                             | + (необязателен, если указан *номер входящий*) |
| employeeCode \| employeePin              | код сотрудника 1С, или его пин       | -                                              |


**Ответ**

> Пин сотрудника - число 1-6 знаков
